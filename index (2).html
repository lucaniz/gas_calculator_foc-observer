Base fee — live & historical
The base fee is the amount of attoFIL (10⁻¹⁸ FIL) burned per unit of gas for every transaction on Filecoin. It fluctuates with network congestion. The current value is fetched live from Glif's public Lotus node. The historical values are computed from actual FWSS on-chain transactions indexed by FOC Observer — they reflect what FWSS users really paid, not theoretical estimates. Minimum floor price — computed from live data
$—
This is the minimum a client must pay per dataset per 30 days to cover the SP's unavoidable daily gas costs. Storage providers must submit two transactions every day for each active dataset — provePossession and nextProvingPeriod — to prove they are still storing the data. These cost gas regardless of what the client is doing.

Formula: (provePossession gas + nextProvingPeriod gas) × base fee / 10¹⁸ × FIL price × 30 days

This value updates automatically when you change the base fee or FIL price above. It is the theoretical minimum — a real floor price should add a margin for gas price volatility.
Dataset & usage parameters
Configure your expected usage pattern. Each parameter maps to on-chain transactions — every transaction costs gas. addPieces calls = how many times you call addPieces in total. Batching = putting multiple pieces in one transaction — it reduces gas per piece because the fixed tx overhead is shared across all pieces in the batch. Datasets = each dataset runs its own daily proving cycle.
