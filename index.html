How costs are calculated
1
Every transaction burns gas. The cost in FIL is: gas_units × base_fee ÷ 10¹&sup8;. The gas_units measure computation. The base_fee is the market price per unit, in attoFIL (10¹&sup8; attoFIL = 1 FIL).
2
Gas units are measured empirically. We queried all FWSS transactions on mainnet since v1.2.0 and averaged the gas used per operation type. For example: provePossession averaged 191.2M gas across 11,134 real transactions. These values are stable because EVM execution is deterministic.
3
Base fee is fetched live from the Filecoin network via Glif's public node. Historical averages (24h, 30d, 90d) are also computed from the same on-chain transaction dataset — they reflect what FWSS users actually paid.
4
Total cost = sum of all operations in the observation period: dataset creation (once), piece additions (per upload), daily proving (every day, per dataset), and termination (once).
Why does dataset size not affect proving cost? The PDP protocol always samples exactly 5 random pieces per proof (hardcoded in the contract), regardless of how many pieces are in the dataset or how large they are. This was verified across all 11,134 provePossession transactions on mainnet — every single one had challenge_count = 5. So a 1 MB dataset and a 1 TiB dataset cost the same gas to prove. The SP's daily proving cost scales only with the number of active datasets, not their size or content.

Usage parameters
Configure your scenario for the observation period below. A few things to be clear about:

Total pieces to add = the total number of individual pieces you plan to add across the whole period.
Avg pieces per addPieces call = how many pieces you put in each transaction on average. If batching is off, this is always 1. If on, you choose a batch size — each call adds that many pieces in one transaction.
Number of transactions is derived automatically: ceil(total_pieces ÷ pieces_per_call).
Proving cost does not depend on dataset size or piece count — it depends only on how many datasets are active. Each dataset requires one provePossession + one nextProvingPeriod per day, always.
